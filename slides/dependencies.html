Bower

  • Command line interface written in Node.js
  • Open source under MIT Licence hosted on GitHub
  • Owned and supported by Twitter
  • Manage a flat tree dependency
  • Runs over Git

bower.json


$ # the "Inception" of packet manager!
$ aptitude install npm && npm install -g bower
$ # create bower.json
$ bower init


{
  "name": "my-project",
  "version": "1.0.0",
  "main": "path/to/main.css",
  "ignore": [
    ".jshintrc",
    "**/*.txt"
  ],
  "dependencies": {
    "name": "version",
    "name": "folder",
    "name": "package"
  },
  "devDependencies": {
    "test-framework-name": "version"
  }
}

The dark side of Bower

  • Definitely a great tool! ... but

  • No convention, just domination!
  • Flat tree dependency is hard to keep consitent
  • Works bad with Grunt when exports are mis configured
  • GitHub is not reliable!
